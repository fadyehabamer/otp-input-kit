otp-input-kit

A highly customizable, framework-agnostic OTP input library with full RTL support, accessibility, i18n, timer, and Web Component.

Zero dependencies ES Module + UMD Full RTL & i18n WCAG 2.1 AA Web Component Dark mode

Installation

📦 NPM
npm install otp-input-kit

Import in your project:

import OTPInput from 'otp-input-kit';
// No separate CSS import needed — it's auto-injected! 🚀
🌐 CDN (UMD)
<script src="https://unpkg.com/otp-input-kit/dist/otp-input.umd.min.js"></script>

Then use global variable:

const otp = OTPInput.create('#container', { ... });

Basic Usage

Choose your preferred way to integrate the library into your project.

🚀 JavaScript API

// 1. Container
<div id="otp-container"></div>

// 2. Initialize
<script>
  const otp = OTPInput.create('#otp-container', {
    length: 6,
    onComplete: (v) => console.log(v)
  });
</script>

🧩 Web Component

<!-- 1. Declare custom element -->
<otp-input length="6" type="numeric"></otp-input>

<!-- 2. Listen for events -->
<script>
  const el = document.querySelector('otp-input');
  el.addEventListener('otp-complete', (e) => {
    console.log('OTP:', e.detail);
  });
</script>

💥 Error Animation Styles
Three built-in error animations — click any button to see them in action. Type a wrong code or use the buttons below.
shake Horizontal wobble — most familiar feedback pattern
highlight Subtle red flash — less intrusive, good for partial errors
both Combines shake + highlight + haptic vibration on mobile
false No animation — just the red border state (a11y-only)
Live validation
Type any code starting with 000 to trigger the error animation automatically.
Try entering "000000" or any code starting with 0
Single-digit shake on invalid character
Numeric-only input — try typing a letter; only that one cell shakes.
// Three error animation modes
OTPInput.create('#otp', { animation: { error: 'shake'     } });  // default
OTPInput.create('#otp', { animation: { error: 'highlight' } });  // red flash
OTPInput.create('#otp', { animation: { error: 'both'      } });  // shake + flash + haptic
OTPInput.create('#otp', { animation: { error: false       } });  // disabled

// Trigger an error animation programmatically (e.g. after server rejects code)
otp.setError('That code is incorrect');

// RTL: shake direction is automatically mirrored to feel natural in rtl layouts.
// Reduced-motion users (prefers-reduced-motion: reduce) get no animation.

Digit Separators

➖ Custom Digit Separators
Visual separators between digit groups — dash, dot, space, or any character
Dash ( 3 + 3 )
Bullet ( 2 + 2 + 2 )
Em dash ( 4 + 4 )
// Dash separator after 3rd digit (3+3)
OTPInput.create('#otp', {
  length: 6,
  separator: { char: '-', after: [3] },
});

// Bullet every 2 digits (2+2+2)
OTPInput.create('#otp', {
  length: 6,
  separator: { char: '·', after: [2, 4] },
});

SMS OTP Auto-Read

📱 Web OTP API — Auto-Fill from SMS
On Android Chrome, the browser reads the OTP from an incoming SMS and fills the inputs automatically
Waiting for SMS… (requires Android + Chrome + HTTPS)
Required SMS format:
Your code is 123456

@yourdomain.com #123456 
// Automatically fills when SMS arrives (Android Chrome + HTTPS)
OTPInput.create('#otp', {
  length: 6,
  smsAutoRead: true,   // uses navigator.credentials.get({ otp: … })
  onComplete: (v) => verify(v),
});

Biometric Confirm

🔐 Biometric Confirm Step (WebAuthn)
After the OTP is accepted, request fingerprint / Face ID / Windows Hello as a second factor
Fill the OTP — then approve the biometric prompt
Checking platform authenticator…
OTPInput.create('#otp', {
  length: 6,
  biometric: {
    enabled: true,
    promptText: 'Verify your identity to continue',
    onConfirmed: () => console.log('Biometric confirmed ✓'),
    onCancelled: () => console.log('Biometric cancelled'),
  },
  onComplete: (v) => {
    // fires only AFTER biometric passes
    verify(v);
  },
});
